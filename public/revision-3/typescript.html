TypeScript es JavaScript con sintaxis para tipos de datos.

Que es TypeScript?

TypeScript (TS) es un lenguaje de programación construido a un nivel superior de JavaScript (JS). Esto quiere decir que TypeScript dota al lenguaje de varias características adicionales que hacen que podamos escribir código con menos errores, más sencillo, coherente y fácil de probar, en definitiva, más limpio y sólido.

Fue creado por Microsoft en 2012 y, desde entonces, su adopción no ha hecho más que crecer. Especialmente, desde que Google decidió adoptarlo como lenguaje por defecto para desarrollar con Angular. Aunque, hoy en día, podemos desarrollar con TypeScript en cualquiera de los frameworks o librerías más punteras, como son React para el frontend o Node para el backend.

TypeScript va un paso más allá de ECMAScript 6 o posterior y añade más funcionalidad a ECMAScript, como tipado fuerte, anotaciones o módulos.

Como TypeScript es un superconjunto de JavaScript, todo el código escrito en JS es válido para TS. Pero no lo contrario. Esto lo podemos comprobar gracias a los navegadores web. Ya que no entienden el código escrito en TypeScript y, para que lo hagan, es necesario hacer una "transpilación" del código a JavaScript.

La transpilación es aplicable hacia cualquier versión de ECMAScript, tanto para navegadores antiguos como para navegadores más actuales.

TypeScript vs. JavaScript

Si profundizamos más en TypeScript y JavaScript, obtenemos una comparativa más exhaustiva de ambos:

  • Tipado estático, genérico, estructural y enumerados: en JS no existen como tal los tipados, aunque los enumerados se pueden simular con clases sencillas. Mientras que TS es un lenguaje fuertemente tipado, donde se pueden crear tipos genéricos o interfaces.
  • Modularización: TS ofrece un soporte directo para módulos, mientras que JS lo hace a través de ECMAScript 6.
  • Tuplas: JS no las soporta, pero si TS.
  • Orientación a objetos: la sintaxis de TS para la programación orientada a objetos es muy similar a la de otros lenguajes como Java o C#. Y además añade clases abstractas y modificadores de acceso, entre otras características. En JS también se puede programar orientado a objetos, pero es algo más complejo.
  • Decoradores: JS no tiene soporte para decoradores, mientras que TS si.
  • Interfaces: Como hemos mencionado, en TS las interfaces son imprescindibles, brindándote la posibilidad de crear escenarios más avanzados. En JS no existe soporte para interfaces.

Instalar Typescript

Primero: es importante que antes de empezar cualquier proyecto con typescript tengas instalado Node.js.



Te mostramos como instalar node

  1. Entrar en https://nodejs.org/es/download/ y descargar el instalador de Node.js en el sistema operativo deseado. Podemos elegir entre Windows, Mac y Linux.

  2. Ejecutar el instalador que acabamos de descargar. Simplemente debemos avanzar en el proceso de instalación.

  3. Una vez finalizado el proceso de instalación, podemos comprobar fácilmente si se nos ha instalado correctamente. Para ello, vamos al intérprete de comandos de nuestro ordenador (en Windows, por ejemplo, escribir “cmd” en la barra de búsqueda y abrir la aplicación de “Símbolo del sistema”).

  4. En la ventana de comandos, escribir node -v y pulsar la tecla Enter. Nos debería aparecer la versión que tenemos instalada de Node.js (en mi caso la versión 12.19.0). Para comprobar que se nos ha instalado también NPM, escribiremos npm -v y pulsaremos de nuevo Enter. Nos debería aparecer también en este caso la versión del Node Package Manager (en mi caso la versión 6.14.8).

  5. Con estos 4 sencillos pasos ya tendremos instalado y disponible para utilizar todo el potencial de Node.js junto con el instalador de paquetes NPM.



Una vez instalado y configurado node en tu ambiente de trabajo Node, ingresamos los siguientes comandos en la terminal para poder inicializar un nuevo proyecto con NPM

              
                //Crea una nueva carpeta
                mkdir primer-proyecto
                cd primer-proyecto
                //Inicializa el nuevo proyecto npm con el siguiente comando
                npm init
                //Inicializa el nuevo proyecto npm con el siguiente comando
                npm install --save-dev typescript tslint @types/node

Segundo: toca crear y configurar la carpeta tsconfig.json.

Cada proyecto de TypeScript debe incluir un archivo llamado tsconfig.json en su directorio raíz. Este tsconfig.json es donde los proyectos de TypeScript definen cosas como qué archivos deben ser compilados, en qué directorio compilarlos y qué versión de JavaScript debe usar.

Se debe crear un nuevo archivo llamado tsconfig.json en la carpeta raíz "primer-proyecto", luego abrirla y copiar el siguiente codigo:

              
                {
                  "compilerOptions": {
                    "lib": ["es2015"],
                    "module": "commonjs",
                    "outDir": "dist",
                    "sourceMap": true,
                    "strict": true,
                    "target": "es2015"
                  },
                  "include": [
                  "src"
                  ]
                }

Tercero: debemos crear tambien la carpeta tslint.json

La creamos ingresando el siguiende comando en la terminal

              ./node_modules/.bin/tslint --init

Cuarto: dentro de nuestra carpeta "primer-proyecto creamos un archivo llamado "index.ts"

Hasta el momento nuestro proyecto quedaria asi:

              
                chapter-2/
                ├──node_modules/
                ├──src/
                │ └──index.ts
                ├──package.json
                ├──tsconfig.json
                └──tslint.json

Primeros Pasos

Luego ingresamos al archivo "index.js" y escribimos el siguiente codigo: 

              console.log('Hello Wordl!')

Y por ultimo, compilamos y corremos el codigo TypeScript ingresando los siguientes comandos en la terminal:

              
                //Compilamos el codigo con TSC
                ./node_modules/.bin/tsc
                //Corremos el codigo con NodeJS
                node ./dist/index.js

Tipos de Datos

Any

En Typescript todo necesita tener un tipo de compilacion, y any es el tipo prederteminado cuando tanto el programador y el typecheker no pueden averiguar de que tipo es una variable. Es un tipo de ultimo recurso, y se debe evitar su uso en lo posible.
¿Por qué deberíamos evitar any?. Un "tipo" es un conjunto de valores y las cosas que podemos hacer con esos valores. any es el conjunto de todos los valores, y podemos hacer cualquier cosa con any. Hace que el valor de la variable se comporte como lo haría en JavaScript, y evita totalmente el typecheker haga su magia. Cuando permitimos any en nuestro código, estámos volando a ciegas. Por eso es que se recomienda usarlo solo como ultimo recurso.

Unknown

Para los pocos casos en los que tenemos un valor cuyo tipo realmente no conocemos de antemano, en lugar de usar any, podemos usar el tipo unknown. Al igual que any, representa cualquier valor, pero en este caso TypeScript no te permite usar una varibale de tipo unknown hasta que no la redefinimos, para poder verificar qué es. ¿Qué operaciones admite unknown? Puede comparar valores desconocidos (con ==, ===, ||, &&, y ?), negarlos (con !) y redefinirlos (como puede hacer con cualquier otro type) con los operadores typeof e instanceof de JavaScript.

Boolean

Los boolean tienen dos tipos de valores: true and false. Podemos compararlos (Con ==, ===, ||, &&, y ?), negarlos (con !), y muchos mas.

                
                  let a = true                  // boolean
                  var b = false                 // boolean
                  const c = true                // true
                  let d: boolean = true         // boolean
                  let e: true = true            // true
                  let f: true = false           // Error TS2322: Type 'false' is not assignable
                                                // to type 'true'.

En este ejemplo podemos ver como funcina:

  1. Podemos dejar que TypeScript deduzca que su valor es un booleano (a y b).
  2. Podemos dejar que TypeScript deduzca que su valor es un booleano específico (c).
  3. Podemos decirle a TypeScript explícitamente que su valor es un booleano (d).
  4. Podemos decirle a TypeScript explícitamente que su valor es un booleano específico (e y f).

Number

Es el conjunto de todos los números: integer, floats, positivos, negativos, infinito, NaN, etc. Los números pueden hacer, bueno, muchas cosas, como sumar (+), restar (-), módulo (%), y comparación (<). Veamos algunos ejemplos:

                
                  let a = 1234                  // number
                  var b = Infinity * 0.10       // number
                  const c = 5678                // 5678
                  let d = a < b                 // boolean
                  let e: number = 100           // number
                  let f: 26.218 = 26.218        // 26.218
                  let g: 26.218 = 10            // Error TS2322: Type '10' is not assignable
                                                // to type '26.218'.
  1. Podemos dejar que TypeScript deduzca que su valor es un número (a y b).
  2. Podemos usar const para que TypeScript deduzca que su valor es un número específico (c)
  3. Podemos decirle a TypeScript explícitamente que su valor es un número (e)
  4. Podemos decirle a TypeScript explícitamente que su valor es un número específico (f y g).

Bigint

Es un tipo recien salido del horno en JavaScript y TypeScript: nos permite trabajar con numeros grandes sin incurrir en errores de redondeo. Mientras que el tipo number solo puede representar números enteros hasta 253, bigint también puede representar números enteros más grandes. bigint es el conjunto de todos los BigInts y admite cosas como la suma (+), la resta(-), multiplicación (*), división (/) y comparación (<).

String

Es el conjunto de todas las cadenas y las cosas que puede hacer con ellas, como concatenar(+), slice (.slice), etc. Veamos algunos ejemplos:

              
                let a = 'hello'                 // string
                var b = 'billy'                 // string
                const c = '!'                   // '!'
                let d = a + ' ' + b + c         // string
                let e: string = 'zoom'          // string
                let f: 'john' = 'john'          // 'john'
                let g: 'john' = 'zoe'           // Error TS2322: Type "zoe" is not assignable
                                                // to type "john".

Object

Los tipos object de TypeScript especifican las formas de los objetos. En particular, es dificil diferenciar la referencia entre objetos simples (como el tipo con {}) y más complicados (del tipo que creas con el nuevo Blah). Esto es por diseño: JavaScript es generalmente tipificado estructuralmente, por lo que TypeScript favorece ese estilo de programación sobre un estilo nominalmente mecanografiado.

Arrays

Al igual que en JavaScript, las matrices de TypeScript son tipos especiales de objetos que admiten cosas como concatenación, pushing, buscar y slicing. Veamos unos ejemplos:

              
                let a = [1, 2, 3]                   // number[]
                var b = ['a', 'b']                  // string[]
                let c: string[] = ['a']             // string[]
                let d = [1, 'a']                    // (string | number)[]
                const e = [2, 'b']                  // (string | number)[]
                let f = ['red']
                f.push('blue')
                f.push(true)                        // Error TS2345: Argument of type 'true' is not
                                                    // assignable to parameter of type 'string'.
                let g = []                          // any[]
                g.push(1)                           // number[]
                g.push('red')                       // (string | number)[]
                let h: number[] = []                // number[]
                h.push(1)                           // number[]
                h.push('red')                       // Error TS2345: Argument of type '"red"' is not
                                                    // assignable to parameter of type 'number'.

La regla general es mantener las matrices homogéneas. Es decir, no mezcles manzanas y naranjas y números en una sola matriz, debemps tratar de diseñar nuestros programas de tal forma que cada elemento de nuestra matriz tenga el mismo tipo. La razón es que de lo contrario, estás tendrá que trabajar más para demostrarle a TypeScript que lo que está haciendo es seguro.

Classes

Las clases son la forma en la que te organizas y piensas sobre tu código, y sirven como unidad principal de encapsulación. Las clases de TypeScript se inspiran mucho en C# y admiten cosas como modificadores de visibilidad, inicializadores de propiedades, polimorfismos, decoradores e interfaces. Pero debido a que las clases de TypeScript se compilan en clases de JavaScript regulares, tambien podemos expresar JavaScript, como mixins de forma segura.

Algunas cosas resumidas que podemos hacer:

  1. Declarar las clases con la palabra clave class. Extenderlas con la palabra clave extends.
  2. Las clases pueden ser concretas o abstractas. Las clases abstractas pueden tener resumen métodos y propiedades abstractas.
  3. Los métodos pueden ser privados, protegidos o, de forma predeterminada, públicos. Pueden ser metodos instanciados o métodos estáticos.
  4. Las clases pueden tener propiedades de instancia, que también pueden ser privadas, protegidas o, por defecto, público. Podemos declararlos como parametros en el constructor o pasar como propiedades de inicializacion
  5. Podemos marcar las propiedades de la instancia como de solo lectura al declararlas.

Interfaces

Cuando usamos clases, normalmente tambien usamos interfaces. Al igual que los alias de tipo, las interfaces son una forma de nombrar un tipo para que no tengamos que definir en línea. Los alias de tipo y las interfaces son en su mayoría dos sintaxis para lo mismo (como expresiones de funciones y declaraciones de funciones), pero hay algunas pequeñas diferencias. 

              
Clase

              
                type Sushi = {
                  calories: number
                  salty: boolean
                  tasty: boolean
                 }
              
              
Interface

              
                interface Sushi {
                  calories: number
                  salty: boolean
                  tasty: boolean
                 }

Conclusion

A lo largo de los últimos años Javascript fue creciendo de manera exponencial, cada vez hay más librerías, más frameworks y más herramientas que nos permiten trabajar de una mejor manera. Originalmente Javascript no fue diseñado para crear aplicaciones de mediana y gran escala, fue diseñado para unas cuantas miles de línea de código, hoy en dia es facil superar millones de lineas de código en un lenguaje de programación dinámico, esto lleva a una serie de problemas a la hora de mantener codigo, especialmente cuando no es nuestro.
Typescript es un lenguaje fuertemente tipado por lo que debemos indicar en todos los casos el tipo de dato a manejar. Permite crear estructuras en base a interfaces de modo que podamos trabajar con una estructura de datos predefinida que no se salga de ciertos parámetros. Ofrece una sintaxis muy similar a la de otros lenguajes como C# o Java, y además añade capacidades inéditas en ECMAScript, como las clases abstractas, modificadores de acceso, tipos enumerados. Las interfaces son esenciales y permiten crear multitud de escenarios avanzados.
Como TypeScript es un superconjunto de JavaScript, todo el código escrito en JavaScript es válido para TypeScript. También se puede transpirlar a cualquier variante de JavaScript ECMAScript que deseemos. Por ejemplo, si sabemos que nuestro proyecto va a trabajar en navegadores antiguos podemos forzar que se compile a ECMAScript 5 (el JavaScript clásico, soportado por todos los navegadores). Sin embargo, si tenemos claro que los usuarios van a emplear navegadores relativamente recientes, podemos generar un código final más sencillo apoyado en ECMAScript 6 o posterior.
En proyectos medianos o grandes, y sobre todo si están involucradas varias personas, escribir el código en TypeScript ofrece grandes ventajas, pero en proyectos pequeños usar Typescript se podría ver como “utilizar una RTX 3090 para jugar al buscaminas” por lo que tal vez no merezca la pena. Existen Frameworks de JavaScript como por ejemplo Angular en donde es obligatorio usar TypeScript ya que fue escrito en ese lenguaje y la documentación oficial promueve el uso del mismo.
TypeScript nos ofrece grandes ventajas, podemos resumirlo en una pequeña lista;
  • Si ya conoces Javascript o las bases de la misma entonces será fácil aprender TypeScript.
  • Convierte a Javascript en un lenguaje equiparable a Java, C#, u otro lenguaje orientado a objetos.
  • Permite estandarizar una estructura de trabajo para que el código sea más fácil de mantener a largo plazo.
  • Da contexto al código, ya que al ser fuertemente tipado el código es más rico en información.
  • Señala errores en el momento de la disposición antes de ejecutar nuestro código.
  • Funciona bien con Frameworks de Front End como Angular, React, Ember y otros más.
Typescript está respaldada por Microsoft, goza de una excelente comunidad, es muy importante tener en cuenta la comunidad a la hora de elegir una herramienta. A día de hoy Angular (un framework en donde es obligatorio el uso de TypeScript) sigue siendo muy demandado en el mercado, por lo cual aprender TypeScript podría ser la opción correcta.
